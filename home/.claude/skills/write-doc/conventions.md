# Document Conventions

## 執筆ルール

- 文体は常体（だ・である調）で統一する。です・ます調と混在させない
- 文章中に不要な改行を含めない（段落内は改行せず、1 段落を 1 行にまとめる）
- 日本語と半角英数字・インラインコードの間に半角スペースを入れる（例: 「React の Hooks」「1 件」）
- リストの行末に句点をつけない
- マークダウンの強調表示（`**`, `*`, `__`）は使用しない
- 専門用語は初出時に定義し、以降は一貫した呼称を使う
- コードの識別子（クラス名・型名・メソッド名など）はインラインコードで囲む
- マークダウンファイルとして出力する

## 文書構成

- 「概要」で始め、資料の目的と対象を最初に述べる
- 本論は簡潔な名詞句の見出しで展開する
- 「まとめ」で要点を再掲して締める
- 導入は内容に応じて「背景」「目的」などに置き換えてもよい
- 比較・対比はテーブルを活用する

## frontmatter 書式

本文の先頭に以下の形式で frontmatter を付与する。

```yaml
---
title: React Hooks
tags:
  - react
  - frontend
  - hooks
updated: 2026-05-23
---
```

| 項目 | 説明 |
| --- | --- |
| title | 資料タイトル（英語。Sentence case。文頭と固有名詞・正式名称のみ大文字） |
| tags | 関連キーワードの配列（ブロック形式） |
| updated | 更新日（`YYYY-MM-DD` 形式） |
