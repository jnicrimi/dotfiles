# describe-pr

## Context

- Open PRs: !`gh pr list --state open --json number,title,author --jq '.[] | select(.author.is_bot == false) | "#\(.number) \(.title)"' 2>&1`

## 概要

既に作成済みのGitHub PRに対して、PR差分からタイトルと本文の候補を生成し、ユーザー承認後に `gh pr edit` で更新する。

Claudeにはブランチのpush権限がないため、PRの作成自体はユーザーが行う。このスキルは「既に存在するPR」のタイトルと本文を整える役割に限定する。タイトル・本文はともに日本語で生成する。

## タイトル規約

Conventional Commits形式に準拠する。

- **MUST**: `type: description` 形式（type は英語、description は日本語）
- **MUST**: 1行目は72文字以内（GitHubのハード上限は256文字だが、本方針では72文字以内を必須とする）
- **SHOULD**: PR全体を要約する単一のまとまった変更を表す

| type | 説明 |
| --- | --- |
| feat | 新機能 |
| fix | バグ修正 |
| refactor | リファクタリング |
| docs | ドキュメント |
| test | テスト |
| style | スタイル（コードの意味に影響しない変更） |
| build | ビルド |
| ci | CI |
| perf | パフォーマンス |
| chore | 雑務 |

## 本文テンプレート

[template.md](./template.md) を参照

## 実行手順

### 1. 対象PRの選択

Contextセクションの `Open PRs` を確認する。

#### 対象PRが0件の場合

`Open PRs` が空、または更新可能なPRが取得できない場合は、以下を出力して処理を終了する。

```text
🤖 PRが見つかりません
────────────────────────────────────────
更新可能なopenのPRがありません。

このスキルは既存のPRを更新します。PRの作成はユーザーの担当です。
PRを作成してから、再度このスキルを実行してください。
```

#### 対象PRがある場合

`Open PRs` を番号付きで提示し、ユーザーに対象PRを選ばせる。

```text
🤖 対象PRの選択
────────────────────────────────────────
1. #42 feat: ユーザー認証機能の追加
2. #41 fix: ログイン不具合の修正

(1-2):
```

- 有効な番号が選択された場合
  - 選択されたPRの番号を記録
- 無効な番号が選択された場合
  - 「無効な番号です。もう一度番号を選択してください。」と再入力を促す

### 2. タイトルと本文の生成・提示

選択されたPRの本文と差分を取得する。

```bash
gh pr view <number> --json body --jq '.body'
gh pr diff <number> --patch
```

- 差分が空（変更ファイルなし）の場合は「PRに変更がありません。」と出力して処理を終了する

#### 2.1 本文の生成

[template.md](./template.md) の固定テンプレートに従い、差分から各セクションを日本語で埋める。

- **MUST**: 本文に既存の内容がある場合は、それを**そのまま保持**し、新しい本文の末尾に残す（上書きしない）
- **MUST**: テンプレートのセクション見出し構成を変更しない

#### 2.2 タイトル候補と本文の提示

タイトル候補（番号付き）と、生成した本文をまとめて提示する。

- **MUST**: 番号が小さいほど推奨度が高い
- **SHOULD**: 簡潔版と詳細版を用意する

```text
🤖 PRの説明を更新
────────────────────────────────────────
📌 対象PR: #[番号] [現在のタイトル]

📝 タイトル候補
────────────────────────────────────────
1. type: 簡潔な説明 (簡潔版)
2. type: より詳細で説明的なタイトル (詳細版)

📄 本文（template.md準拠）
────────────────────────────────────────
[生成した本文をそのまま表示]

タイトル番号を選択してください。本文はこの内容で更新します。
(1-2):
```

- 有効な番号が選択された場合
  - 選択されたタイトルを記録
- 無効な番号が選択された場合
  - 「無効な番号です。もう一度番号を選択してください。」と再入力を促す
- 本文に修正要望があれば反映して再提示する

### 3. PRの更新

選択されたタイトルと生成した本文で、既存PRを更新する。

- タイトルが72文字以内であることを検証する。超過する場合は短縮版を再提示する
- 本文は `--body-file -` で標準入力から渡し、シェルのクオート崩れを防ぐ

```bash
printf '%s' "[本文]" | gh pr edit <number> --title "[タイトル]" --body-file -
```

### 4. 完了報告

更新したPRの番号・タイトル・URLを提示する。

```text
🤖 更新完了
────────────────────────────────────────
📌 #[番号] [新しいタイトル]
🔗 [PRのURL]
```
